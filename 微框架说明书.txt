====================================================================
Mulberry (桑葚) 代理微框架 - 核心开发者说明书与架构指南
(阶段：空间拓扑引擎 & 节点族群编组版)

一、 项目基本情况概述

Mulberry 是一个基于"有向无环图 (DAG)"原则构建的、面向多 Agent 协同与上下文管理的轻量级可视化执行微框架。

本项目的核心思想是"拓扑即逻辑"：通过前端画布上的节点连线，动态定义大语言模型 (LLM) 的身份、记忆上下文以及执行流向。项目遵循极简 MVP（最小可行性产品）原则，摒弃了繁重的 ORM 型数据库和过度封装的第三方库，赋予开发者绝对的底层控制权。

【最新架构演进：空间拓扑引擎 & 节点族群编组】

布局重构：彻底抛弃 Flex 平铺切割排版，采用"中心画布 100% 铺底 + 侧边栏/小地图 Absolute 绝对定位悬浮"的 Z 轴层叠架构，实现了画布节点可丝滑穿透底层 UI 的全景沉浸式工作台。
渲染引擎：弃用写死的 Tailwind 静态色值类，跨越 React 生命周期，采用 var(--uop)（上层视图组透明度）与 var(--nop)（节点视图组透明度）两大原生 CSS 变量接力渲染。实现了百帧级流畅拖动、无重绘滞后的实时光学混合与毛玻璃 (Glassmorphism) 视效。
双轨持久化引擎：零数据库设计。系统配置（Excel 数据源路径、两套独立透明度偏好）与拓扑图纸工程（节点/连线坐标与内容）均能实现向本地 .json 的静默防丢档物理直写。
空间编组引擎：引入"集合 (CollectionNode)"范围容器节点，实现了不依赖连线的空间父子归属体系。子节点可通过手动拖入/拖出范围框动态建立或解除隶属关系，框内子节点随父框联动移动。系统采用自研的递归父链绝对坐标回溯算法和 AABB 中心点碰撞检测机制，完全绕过 React Flow 内部坐标缓存，确保拓扑变更时落点精准。
准星粘贴引擎：Ctrl+V 粘贴操作精确锚定于当前鼠标指针在画布中的位置，取代传统的原地偏移粘贴。内置连续粘贴防重叠阶梯步长补偿机制，同时保留复制父子族群时的拓扑血缘映射。
节点品类扩充：在原有角色体系（角色提示词、角色块、内容块、调试标记、开始聊天按钮）基础上，新增工具块 (ToolNode，浅紫/深紫)、模型调用情况 (ModelStatusNode，浅蓝/深蓝)、集合 (CollectionNode，浅灰/深灰) 三种节点类型。
拓扑排序保障：引入 sortNodesForParentPriority() 拓扑排序函数，确保集合框父节点在节点数组中始终先于其子节点排列，满足 React Flow 引擎对父子声明顺序的强制约束。

【技术栈组成】
运行环境：Windows 11 / Node.js / Python 3.12
前端体系：Vite + React (纯 JavaScript) + @xyflow/react (图编译核心，含 NodeResizer 形变插件) + Tailwind CSS (UI 样式)
后端体系：FastAPI + httpx (高并发流式请求) + openpyxl (轻量 Excel 解析)
通信机制：HTTP RESTful + Server-Sent Events / SSE (流式打字机响应) + CSS Custom Properties (纯视觉帧渲染)

二、 项目物理结构与功能映射字典（核心修改指南）

Mulberry 采用前后端分离的"单体仓库 (Monorepo)"结构。前端实现了 UI 视图层、业务逻辑层与主事件栈的深度职能分离；后端兼负轻量级文件 I/O 调度总线的职责。

│ - ToolNode (工具块) 浅紫/深紫 (#581c87)，含行内双击编辑

│ - ButtonNode (开始聊天按钮) 琥珀渐变，触发 mulberry-start-chat 全局事件

│ - 开始聊天按钮 (buttonNode)

│ └── FlowUI.jsx [独立视图组件层] 收容时钟、设置模态验证层、透明度双滑块调节器及右侧控制版面。全面接入 var(--uop)。
├── hooks/
│ └── useFlowEngine.js [核心业务网络层] 收聚 SSE 管道、Fetch 通信请求及图谱推拉状态栈。
└── App.jsx [主应用装配总管(重要)] 实现 Z 轴绝对定位排版，负责基础画布挂载、热键监听、越权越级覆写三方库样式的入口，并承载以下核心引擎：
  - 节点类型注册中心 (nodeTypes 字典)

四、 空间编组引擎行为规约（集合节点交互规则）

1. 集合节点 (CollectionNode) 是一种可自由调整大小的透明范围框，在画布中没有名字，仅在侧边栏工具栏中显示为"集合"。
2. 集合节点视觉始终置于最下层 (zIndex: -1)，以便其他节点叠放在其上方。
3. 仅当用户手动将普通节点拖入集合节点范围内并松手时，才会建立父子关系。拖拽集合节点去碰撞普通节点不会产生任何效果。
4. 集合节点与集合节点之间无法形成父子关系，互相碰撞没有作用。
5. 父子关系建立后，子节点将随父集合节点联动移动。
6. 用户需手动将子节点从集合节点内拖出并在外部松手，方可解除父子关系。
7. 若将已归属于集合 A 的子节点拖入集合 B 中松手，父子关系将转移至集合 B。
8. 若拖动含有子节点的集合 A 去碰撞集合 B，集合 A 内的子节点归属不变。
9. 碰撞检测算法：以被拖拽节点的中心点坐标进行 AABB 包围盒判定，命中目标为所有集合框中反向遍历命中的第一个。
10. 坐标系转换：加入父框时，世界绝对坐标自动转为相对于父框左上角的本地坐标；脱离父框时，递归回溯父链累加还原为世界绝对坐标。

五、 扩展需求对应指南 (Where to Edit)

【场景 1：增加一种全新的节点种类】
第一步：在 src/CustomElements.jsx 编写新的节点组件，必须使用 bg-[rgba(R,G,B,var(--nop,0.85))] 语法对接系统透明度，并挂载 backdrop-blur-sm；
第二步：在 src/Sidebar.jsx 的 templates 数组中增加该节点字典，classes 属性需保持严格一致；
第三步：在 src/App.jsx 顶部 nodeTypes 字典中注册，于 onDrop 中配置其初始生成姿态（含默认 data 字段），并在 getMiniMapNodeColor 中补充小地图追踪色。

【场景 2：增删或修改全局悬浮 UI（如面板、时钟、设定菜单）】
修改点：打开 src/components/FlowUI.jsx。
此类组件已接入 var(--uop) 渲染流。若要增加新悬浮框，需在其 className 中注入如 bg-[rgba(255,255,255,var(--uop,0.70))] backdrop-blur-md 等滤镜以保持全应用毛玻璃质感统一。

【场景 3：修改大语言模型 (LLM) 的调用逻辑或接入新的 API 供应商】
修改点：打开 mulberry-backend/main.py。
定位到 _chat_generator 函数内。根据 openpyxl 提取的数据字典，在 httpx 块中重构 Base URL、报文组装。

【场景 4：修改数据落盘（配置项扩充或持久化规则）】
修改点 1：后端打开 main.py，于 save_config 路由内追加新的 JSON 键值拦截。
修改点 2：前端若需暴露该设置，在 FlowUI.jsx 的 SettingsModal 中增加对应滑块或输入框，并在 Fetch 请求体中补传。

【场景 5：干涉节点拖拽、连线生成、双击编辑的总规则】
修改点：基础鼠标动作和 React Flow 原生响应落在 App.jsx（包含 onNodeDragStop 空间编组归属探测引擎、onDrop 空投拦截器、键盘快捷键准星粘贴引擎）。具体的节点内行内编辑 (InlineEditableArea) 及状态切换的双击事件 (onDoubleClick) 落在 CustomElements.jsx 并在此派发 mulberry-action-settled 保存指令。

【场景 6：修改空间编组归属逻辑（父子关系检测规则）】
修改点：打开 src/App.jsx，定位到 onNodeDragStop 回调函数。
碰撞检测逻辑位于此回调内部。归属判定依赖 getAbsolutePosition() 递归回溯函数计算节点的世界绝对坐标，并通过 AABB 中心点包围盒算法判定是否落入某个 collectionNode 的矩形区域。
拓扑变化后由 sortNodesForParentPriority() 执行父先子后排序以满足 React Flow 渲染引擎约束。

六、 生产/开发环境搭建与部署流 (针对 Windows 11)

【环境准备阶段】

确保已安装 Node.js (推荐 v20 LTS 或以上)与 Python 3.12。
确保在系统环境变量中事先配置了所需的 LLM 凭证（例如：新建 DEEPSEEK_API_KEY）。
准备符合标准表头的 角色配置.xlsx 文件。
【启动顺序规约】

终端切入后端：cd mulberry\mulberry-backend
唤醒沙盒：.\venv\Scripts\activate
启动 ASGI 总线：uvicorn main:app --reload
另起新终端切入前端：cd mulberry\mulberry-frontend
启动 Vite 热更总控：npm run dev
访问日志链接（http://localhost:5173） 。
【双轨引擎操作规约与持久化行为鉴权】

环境变量/透视率自动固化：点击右上角齿轮，修改路径或拖拽透明度滑块。验证绿灯或鼠标/触控松开 (onMouseUp) 时，系统将静默执行 HTTP POST 向硬盘覆写并锁档。冷启必定恢复。
画布拓扑手动固化：画布极大的数据吞吐量由人为下发指令管理。由于项目含内存保护机制，拖拽、修图后立刻按 [Ctrl+Z] 支持毫秒级内存回溯；确认节点拓扑无误后，必须手动点击右上角"保存（深绿按钮）"进行物理封装防丢档。
